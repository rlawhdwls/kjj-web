박스 모델(Box Model)

CSS는 요소를 표시하기 위해 요소별로 사각형의 박스를 생성한다. CSS3에서는 속성이 다향해졌지만, 요소를 사각형 박스로 표시하는 것은 크게 다르지 않다. HTML 요소 박스의 크기는 컨텐츠 크기에 안쪽 여백인 padding과 테두리인 border 그리고 바깥쪽 여백인 margin에 의해 결정된다.

  • content - 요소의 컨텐츠가 표시되는 영역
  • padding - content 영역과 border 사이의 안쪽 여백
  • border - padding 영역과 margin 사이의 테두리
  • margin - border를 기준으로 다른 요소와의 바깥쪽 여백

display

display는 HTML 요소의 표현 방식을 지정하는 속성으로, HTML요소는 각각 기본으로 지정된 display값이 존재하며, 그 값도 다양하다.
display 속성 값이 inline으로 지정된 요소는 자신의 전후로 줄바꿈을 만들지 않으며, 포함하는 요소가 너비에 의해 자동 줄바꿈된다. 요소의 폭은 컨텐츠에 따라 결정되며, width 속성으로 요소의 너비를 직접 지정할 수는 없다.
block으로 지정된 요소는 기본적으로 부모 요소의 너비 전체를 채우며, 요소 박스 전후로 줄바꿈이 생긴다.
inline-block으로 지정된 요소는 iunline과 같이 요소 전후로 줄바꿈이 생기지는 않지만, block요소처럼 width 속성으로 요소의 너비를 직접 지정할 수 있다.
none값은 화면에 요소를 표시하지 않는다.
run-in은 요소의 형제 요소가 block 요소가 아니면 ,block 요소로 정의되고, block요소이면 inline 요소로 정의된다.
list-item은 목록 요소로 보여지도록 한다. 그밖에 table, ruby가 있다.

  • content - 요소의 컨텐츠가 표시되는 영역
  • padding - content 영역과 border 사이의 안쪽 여백
  • border - padding 영역과 margin 사이의 테두리
  • margin - border를 기준으로 다른 요소와의 바깥쪽 여백

overflow

overflow는 컨텐츠가 요소의 컨텐츠 영역을 벗어나는 경우의 처리 방법을 지정하는 속성으로, visible은 오버 플로우된 컨텐츠를 그대로 표시한다. 이때 오버 플로우된 컨텐츠의 영역은 요소의 크기에 반영되지 않는다. hidden의 경우 오버 플로우된 컨텐츠를 숨기고 scroll은 요소 박스에 스크롤 바를 생성한다. auto 속성 값의 경우 사용자 도구의 기본 설정 값대로 작동하도록 한다.

hidden - overflow는 콘텐츠가 요소의 콘텐츠 영역을 벗어나는 경우 처리방법을 지정하는 속성이다. overflow 속성을 방향 별로 지정하기 위한 상세 속성 overflow-X, overflow-Y를 사용 할 수 있다.

scroll - overflow는 콘텐츠가 요소의 콘텐츠 영역을 벗어나는 경우 처리방법을 지정하는 속성이다. overflow 속성을 방향 별로 지정하기 위한 상세 속성 overflow-X, overflow-Y를 사용 할 수 있다.

overflow-y - overflow는 콘텐츠가 요소의 콘텐츠 영역을 벗어나는 경우 처리방법을 지정하는 속성이다. overflow 속성을 방향 별로 지정하기 위한 상세 속성 overflow-X, overflow-Y를 사용 할 수 있다.

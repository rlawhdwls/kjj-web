10.폼관련 요소


HTML5 에서는 다수의 폼 관련 요소가 추가 되었습니다. HTML5에서 추가된 폼 관련 요소는 입력 값 후보를 나타내는 datalist 요소,
폼의 계산 결과나 input 요소의 range type의 출력 값을 표현하는 output 요소, 폼을 전송할 때 공개 키나 비밀 키를 생성하는 keygen 요소,
진행 상황을 표시하는 progress 요소, 한정된 범위의 값을 나타내는 meter 요소 등이 있다.

폼form

form 요소는 서버에 사용자 입력 값을 전송하는 역할을 담당하는 요소이다. form 요소에는
기존에 제공하던 속성 이외에 accept-charset 속성, autocomplet 속성, novalidate 속성이 추가되었다.
    form요소에 사용할 수 있는 속성
  • accept-charset > 폼 전송에 사용할 문자 인코딩 지정(문자 인코딩 값을 둘 이상 지정 가능하며, 이때 문자 인코딩 값을 공백으로 구분하여 선언)
  • action > 폼 데이터를 전송할 곳의 URL
  • autocomplete > 자동 완성 기능의 활성화 여부(on, off 지정)
  • enctype > 폼 전송 데이터의 MIME 타입
  • method > 폼 전송에 사용할 http 메소드(get 또는 post)
  • name > 폼의 이름
  • novalidate > 폼 전송 시 입력 데이터 값의 검증을 수행하지 않음(논리 속성)
  • target > 폼의 전송 값을 처리할 대상

폼 서식 그룹fieldset, legend

fieldset 요소는 폼 요소 안에 관련 있는 폼 컨트롤들을 그룹화할 때 사용한다. 하위에 legend 요소를 사용하여 그룹화한 폼 컨트롤의 목적이나 이름을 마크업할 수 있다.
HTML5에서 fieldset 요소에 새롭게 추가된 속성인 form 속성을 사용하면 form 요소 안에 반드시 fieldset 요소를 포함시키지 않아도
해당 form 요소와 연결할 수 있다. 연결하는 방법은 form 요소에는 id 값을 할당하고, fieldset 요소에는 속성으로 form을 지정하며, 해당 속성 값으로
연결하고자 하는 form 요소의 id 값을 지정한다.

서식 레이블label

웹 접근성 측면에서도 중요한 요소로, 다양한 폼 서식의 설명을 의미한다.
가령 아이디 입력 상자나 비밀번호 입력 상자의 경우, 기계가 해당 입력 상자가 어떤 데이터를 입력하는 영역인지 알 수 없기 때문에
label 요소로 연관 관계의 의미를 설명할 수 있다.
이렇게 폼 서식의 연관성을 명확하게 지정하면 보조 기기 등에서 폼 서식의 접근이나 이해가 쉬워진다.

label 요소로 연관 관계를 설명할 수 있는 폼 서식으로 button, input, keygen, meter, output, progress, select, textarea 요소들을 들 수 있다.

label 요소를 사용하여 폼 서식과의 연관 관계를 설명할 때는 for 속성을 사용할 수 있다. for 속성에서는 연결하고자 하는 폼 서식의 id속성 값을 지정해야 하며,
fieldst 요소와 마찬가지로, 새로 추가된 form 속성도 사용할 수 있다.

다양한 폼 서식input

input 요소는 여러 가지 폼 서식을 마크업할 때 사용한다. 특히 HTML5에서는 기존에 제공하던 10가지 type 이외에 search, tel, url 등 다수의 type이 새롭게 추가되었다.
type 속성 이외에 autocomplete 속성, list 속성, required 속성, maxlengh 속성, pattern 속성, max 속성, min 속성, step 속성, placeholder 속성, multiple 속성, maxlength 속성
autofocus 속성 등 다수의 신규 속성도 추가되어 성능이 한층 더 강화 되었다.
그러나 많은 신규 기능이 아직 모든 웹 브라우저에서 지원되고 있지 않으므로 사용시 웹 브라우저에서 해당 기능을 제공하는지 확인해야 한다.

input 요소는 type 속성 값에 따라 폼 서식의 종류가 결정된다.

input type 속성명 설명 결과
type = "text" text 타입은 한 줄 글상자를 나타낼 때 사용한다.
type = "password" password 타입은 한 줄 글상자와 렌더링 결과가 동일하지만,
입력 값을 화면에 출력하지 않는 형태로 나타내기 때문에
주로 클라이언트 환경에서 해당 값을 보이지 않도록 할 때 사용
type = "search" search 타입은 text와 별 차이 없지만, 줄바꿈을 포함한 문자열 값이
입력될 경우 줄바꿈을 제거한다.
search 타입의 원래 목적은 text타입과 구별하여 표시하기 위함이다.
type = "tel" tel 타입은 text타입과 동일하며 차이점은 입력 값이 전화번호를 위한것이다.
또한 줄바꿈을 포함한 문자열 값이 입력될 경우 줄바꿈을 제거한다.
모바일 등의 디바이스에서 해당 서식에 값을 입력할 때 숫자 입력 모드로 제공된다.
type = "url" url 타입은 한 줄 글상자와 동일하며 입력 값이 URL을 위한 것임을 의미한다.
줄바꿈을 포함한 문자열이 들어올 경우 웹 브라우저가 이를 제거한다.
입력될 수 있는 값은 절대 경로 URL만 가능하다.
type = "email" email 타입은 한 줄 글상자와 동일하며, 입력 값이 이메일을 입력하기 위한 것이다.
이메일 주소만 입력 가능하며, 부적절한 이메일 주소가 입력될 경우 에러 메시지를
전송한다. 모바일의 경우 영문 및 숫자 입력 모드로 변경된다.
type = "date" 날짜를 입력하고자 할 때 사용.
date 타입을 사용하면 웹 브라우저에서 날짜를 입력할 수 있는 입력 폼을 함께 제공한다.
min, max 속성을 사용하여 입력 날짜의 범위를 지정할 수도 있다.
step 속성을 사용하면 최소 단위를 변경할 수 있다.
type = "month" date 타입과 동일하지만, 날짜가 아닌 월을 입력하고자 할 때 사용한다.
사용할 수 있는 속성도 date 타입과 같다.
type = "week" date 및 month 타입과 동일하며, 날짜나 월이 아닌 주 단위로 입력하고자 할 때 사용한다.
type = "time" time 타입은 시. 분. 초. 등의 시간을 입력하고자 할 때 사용한다. 사용할 수 있는 속성도 date 및 month, week 타입과 같다.
type = "datetime" 날짜와 시간을 입력하고자 할 때 사용한다. 입력한 일시의 타임 존을 UTC로 설정하기 때문에
한국 시각과 비교하면 9시간 늦은 일시가 된다. 서버 쪽에서 datetime 값을 처리할때는 이점을 염두 해야 한다.
datetime 타입의 경우 date, month, week, time 타입과 마찬가지로 min, max 속성을 사용하여
입력 값의 범위를 지정할 수 있다. 또한 step 속성을 사용하면 최소 단위를 변경할 수 있으며 일시를
나타내는 최소 단위는 60초로 설정되어 있다.
type = "datetime-local" 타임 존이 없는 날짜와 시간을 입력하고자 할 때 사용한다.
타임존이 없다는 것을 제외하고는 datetime 타입과 동일하다.
type = "number" 숫자를 입력하고자 할 때 사용한다. max, min 속성을 사용하여 최소값과 최대값을 지정할 수 있으며, step 속성을 사용하여 입력
단위를 지정할 수도 있다.
type = "range" 숫자를 입력하고자 할 때 사용한다. 동일하게 숫자를 입력할 수 있는 number타입과의 차이점은
슬라이더 형태의 UI로 렌더링된다는 것입니다. number 타입과 마찬가지로 min, max, step 속성을 함께 사용하여
최대, 최소 및 입력 단위를 지정할 수 있다.
type = "color" 색상을 입력하고자 할 때 사용한다. color타입은 아직 모든 웹 브라우저가 지원하고 있지 않지만,
일부 웹 브라우저에서 Color Picker 형태의 UI로 렌더링된다.
type = "checkbox" 체크 박스 입력 서식을 제공하고자 할 때 사용한다. 논리 속서인 checked 속성을 지정하면
해당 체크 박스가 미리 선택된 상태로 렌더링 된다.
type = "radio" 라디오 버튼 입력 서식을 제공하고자 할 때 사용한다. checkbox타입과 마찬가지로 radio 타입에
논리 속성인 checked 속성을 지정하면 해당 라디오 버튼이 미리 선택된 상태로 렌더링된다.
type = "file" 로컬의 파일을 업로드할 수 있는 서식을 제공하고자 할 때 사용한다. file 타입에 accept속성을 사용하여
서버 쪽에서 수신할 수 있는 파일 형식을 지정할 수 있다.
논리 속성인 multiple속성을 사용하여 여러 개의 파일을 선택하도록 할 수 있다.
type = "hidden" 웹 브라우저 화면에 나타나지 않으며, 자바스크립트를 이용하지 않으면 내용을 변경할 수 없는 숨김 서식이다.
hidden 타입의 name 속성에 _charset_값을 지정하면, 해당 폼의 문자 인코딩 값이 서버로 전달된다.
type = "submit" 전송 버튼을 나타낸다. 이 버튼의 역할은 폼 데이터를 서버로 전송하는 것이며, value 속성의 버튼에
표시될 문자열을 입력할 수 있다.
type = "reset" 리셋 버튼을 나타낸다. 이 버튼은 폼 데이터를 모두 초기화하는 기능을 가지고 있으며, value 속성의 버튼에
표시될 문자열을 입력할 수 있다.
type = "button" 단순 버튼으로 자바스크립트 등을 실행하기 위한 용도로 사용된다. submit 타입이나 reset 타입과 마찬가지로
value 속성의 버튼에 표시될 문자열을 입력할 수 있다.
type = "image" submit타입과 마찬가지로 전송 버튼을 나타낸다. submit 타입과 다른 점은 버튼에 사용할 이미지를 지정할 수있다는 것이다.
버튼 이미지 경로는 src속성으로 지정하며, alt 속성을 사용하면 해당 버튼 이미지의 대체 텍스트를 지정할 수 있다.
image타입에서는 src 속성과 alt속성이 필수 속성입니다.


input 속성 설명 결과
autocomplete 속성 자동 완성 기능을 사용할 것인지를 결정하는 속성으로 on과 off값을 지정할 수 있다.
on값을 지정한 경우 이전에 입력한 데이터를 웹 브라우저에서 입력 후보 값으로 제공한다.
list 속성 웹 개발자가 직접 입력 후보 값을 제공하는 기능으로, autocomplete 속성과 비슷하다. 이때 입력 후보 값은 datalist 요소를 사용하여 마크업한다.
readonly 속성 읽기 전용으로 지정하기 위해 사용하는 논리 속성이다. 해당 서식을 readonly로 지정하면 사용자가 값을 변경할 수 없다.
size 속성 화면에 보이는 폼 서식의 크기를 지정하는 속성이다. 이때 웹 브라우저에 따라 전각 글자 크기와 반각 글자 크기 중 어느 것을 기준으로 하느냐에 따라 폼 서식의 크기가 달라질 수 있다. 이러한 이유로 폼 서식의 너비를 정확하게 지정하고자 한다면 CSS를 이용하는것이 좋다.
required 속성 해당 폼 서식이 필수 속성인지 아닌지를 나타내는 논리 속성이다. required 속성을 지정한 폼 서식에 값을 입력하지 않은 상태로 서버로 데이터를 전송할 경우, 에러 메시지가 웹 브라우저 화면에 출력된다.
placeholder 속성 사용자에게 해당 폼 서식에 어떤 값을 입력해야 하는지를 알리기 위한 힌트로 사용된다. 이때 placeholder 속성에 입력한 값은 해당 폼 서식에 포커스가 있을 때 새로운 값을 입력하면 사라진다.
pattern 속성 폼 서식에 입력하는 입력 값의 정규 표현식을 의미한다. 정규 포현식으로 입력 값의 혀식을 제한할 수 있으며, 이때는 title 속성을 사용하여 정규 표현식에 대한 설명을 함께 제공하는 것이 좋다.
multiple 속성 논리 속성으로 2개 이상의 값을 지정할 때 사용한다. 새로운 email타입과 file타입에 사용할 수 잇다.
maxlength 속성 해당 폼 서식에 입력할 수 있는 최대 글자 수를 지정할 때 사용한다. maxlengh 속성은 한글과 영문을 구별하지 않고 각각 한 문자로 계산한다.
autofocus 속성 웹 페이지가 로딩되자마자 바로 폼 서식으로 포커스가 이동하도록 하는 논리 속성이다.
step 속성 폼 서식에 지정할 수 있는 값의 단위이다. 예를 들어 number 타입에 step="5"0,5,10---형태로 값이 입력된다. step값은 input 요소의 type 값에 따라 단위가 달라진다.
max, min 속성 입력 최대값 max, 입력 최소값 min
disabled 속성 논리 속성으로, 해당 폼 서식을 사용할 수 없음을 의미한다.
name 속성 폼 서식의 이름을 의미하며, 폼 데이터를 서버로 전송할 때 해당 데이터를 가리키기 위한 이름으로 사용한다.
form 속성 input 요소가 나타내는 폼 서식을 지정한 form 요소와 연결할 때 사용한다. 기존 HTML4.01이나 XHTML1.0에서 모든 폼 서식은 form요소 안에 마크업해야한 했다. 그러나 HTML5에서는 form 속성의 추가로 인해 폼 서식을 form 요소안에 마트업하지 않고도 연결하는 것이 가능해졌다.


버튼button

버튼 서식을 삽입할 때 사용한다. button 요소에 type 속성을 지정하지 않으면 해당 버튼이 전송 버튼이 된다. button 요소의 type 속성에 지정할 수 있는 값은 submit, reset, button 등이며, input 요소의 submit, reset, button과 기능이 동일하다.

목록서식select, optqroup, option

select 요소는 항목을 선택할 수 있는 폼 서식을 마크업할 때 사용한다. select 요소에서 지원하는 size 속성은 한번에 표시할 항목의 수를 의미하며, mutiple 속성은 다중 선택을 허용할 것인지의 여부를 결정하는 논리 속성이다. optgroup 요소는 select 요소 안에서 사용할 수 있는 그룹 요소로, 각 항목을 의미하는 option 요소를 그룹화하여 지정할 때 사용한다. optgroup 요소의 경우, label 속성을 사용하여 해당 그룹의 역할을 이해할 수 있도록 제공할 수 있다.

입력 값 후보datalist

HTML5에 새롭게 추가된 요소로, 텍스트 상자의 입력 값의 후보 목록을 그룹화할 떄 사용한다. 기존HTML4.01이나 XHTML1.0에서 자바스크립트 등을 사용하여 구현했지만, 이제부터는 자바스크립트 없이 이러한 구현이 가능하도록 마크업상에서 지원한다.

여러 줄 글상자 textarea

여러 줄 글상자의 의미로, 텍스트 콘텐츠를 입력받기 위한 폼 서식이다. 해당 요소내에서는 텍스트의 줄바꿈도 가능하다. textarea 요소에 새롭게 추가된 속성으로는 placeholder, required 속성을 들 수 있다. 요소의 wrap 속성은 textarea 요소 안에서 자동 줄바꿈 처리된 텍스트에 줄바꿈 명령ㅇ을 넣을 것인지의 여부를 결정하는 속성으로, 기본 값은 soft입니다. 줄바꿈을 넣지 않겠다는 의미로, hard는 줄바꿈을 넣겠다는 의미로 사용한다.



키쌍 생성keygen

폼을 전송할 때 키를 생성하는 서식으로, 전자 인증에 주로 사용한다. keygen 요소를 사용하면 폼이 전송된 후에 비밀 키와 공개 키를 생성하여 비밀 키는 웹 브라우저에 저장되고 공개키는 서버에 전송한다. HTML5에서 keygen 요소에 지원하는 keytype은 rsa만 가능하다. keygen 요소를 사용하면 웹 브라우저에서 RSA 전자 서명 알고리즘에 사용할 암호화 비트수를 선택할 수 있다.

진행 상황 progress

작업의 진행 상황을 나타내고자 할 때 사용한다. HTML4.01이나 XHTML1.0에서는 진행 상황에 대한 의미를 마크업 할 수 있는 요소가 존재하지 않았지만, HTML5에서 추가된 progress 요소를 사용하면 마크업으로 진행하는 상황이라는 의미를 부여할 수 있다.

한정된 범위 값 meter

meter 요소는 progress 요소와 혼동할 수 있는 요소로, 작업의 진행 상태가 아닌 한정된 범위의 값을 나타내고자 할 때 사용한다. 한정된 범위의 값을 의미하는 meter 요소를 사용하여 마크업할 때는 max 속성의 경우 최대값을, min 속성의 경우 최소값을 지정할 수 있다.

    meter 요소에서 사용할 수 있는 속성
  • value - 값
  • min - 최소값
  • max - 최대값
  • low - 낮은 영역의 상한 값으로 지정하지 않으면 min 속성 값과 같은 값으로 간주
  • high - 높은 영역의 상한 값으로 지정하지 않으면 max 속성 값과 같은 값으로 간주
  • optimum - 최적 값으로 지정하지 않으면 amx 속성과 min 속성의 중간 값으로 간주

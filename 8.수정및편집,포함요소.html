추가ins

ins 요소는 추가로 삽입된 컨텐츠를 의미한다. 해당 컨텐츠가 문서에 추가되었음을 표현할때 사용한다. 고유 속성으로 cite와 datetime 속성을 가지며, cite 속성은 해당 컨텐츠를 추가하는 이유나 근거가 되는 URL을 명시할 수 있다. datetime 속성을 사용하면 컨텐츠를 추가한 날짜 및 시간을 지정할 수 있다.

그러나 <hgroup> 요소는 현재 HTML5 사양에서 삭제된 상태입니다.

삭제del

del 요소는 삭제된 컨텐츠를 의미하며, 해당 컨텐츠가 문서에서 삭제되었음을 표현할때 사용한다.
고유 속성으로 cite와 datetime 속성을 가지며 cite 속성은 해당 컨텐츠를 삭제하는 이유나 근거가 되는 URL을 명시한다.
datetime 속성을 사용하면 컨텐츠를 삭제한 날짜 및 시간을 지정할 수 있다.

포함 요소

이미지img

이미지 컨텐츠를 삽입하고자 할 때 사용하는 요소로, 이미지 컨텐츠의 경로는 필수 속성인 src 속성에 명시한다.
이때 이미지 형식 이외에 pdf파일이나 svg파일도 지정할 수 있다.
scr 속성 이외에 img 요소에서 중요한 속성으로는 alt 속성을 들 수 있다.
alt 속성은 대체 텍스트의 의미로, 웹 접근성의 관점에서 이미지가 표시되지 않거나 이미지를 볼 수 없는 환경에서 alt속성만으로도 이미지의
내용을 사용자가 이해할 수 있도록 명확하게 제공해야 한다. 또한 alt 속성은 문법상 필수 속성이므로 생략하면 안된다.

인라인 프레임iframe

HTML5에서 유일하게 폐지되지 않은 프레임 요소이다. src 속성을 사용하면 별도의 HTML 문서를 삽입할 수 있다.
HTML5에는 src속성 이외에 HTML 마크업 코드 자체를 포함할 수 있는 srcdoc 속성이 추가되었다.
만약 srcdoc 속성과 src 속성을 함계 사용할 경우 srcdoc 속성이 우선한다. 이때 srcdoc 속성을 지원하지 못하는 웹 브라우저에서는
src 속성이 동작한다. 또한 새롭게 추가된 sandbox 속성의 경우에는 프레임에 포함할 컨텐츠에 제약을 걸 수 있으며 , seamless 속성은 프레임 영역의 테두리와 스크롤 막대가 렌더링되지 못하게 할 수 있다.
그러나 아직까지 srcdoc 속성과 sandbox, seamless 속성을 지원하는 웹 브라우저는 없다.

객체 삽입embed

플러그인 관련 컨텐츠를 삽입할 때 사용하는 요소로, HTML5에서 정식 사양에 포함된 요소이다.
embed 요소는 빈 요소이기 때문에 종료 태그를 기술하면 안된다. src, type, width, height 속성 이외의 독자적인 속성은 네임 스페이스를 사용하지 않으면 인정되지 않는다.

객체 삽입object, param

    object
  1. 이미지, 플래시, 동영상, 비디오 등 이미지 컨텐츠나 플러그인 컨텐츠를 삽입할 때 사용한다.
  2. 플러그인 컨텐츠 지정은 data 속성에, MIME 타입은 type 속성에 지정한다.
  3. 지정한 컨텐츠의 URL에서 컨텐츠를 가지고 오지 못하는 경우 또는 해당 컨텐츠를 지원하지 못하는 환경(예: 플래시)이 존재하는 경우에는 object 요소의 시작 태그와 종료 태그 사이에 폴백(fallback)컨텐츠를 제공할 수 있다.
    param
  1. 플러그인의 기본 값으로 사용할 파라미터를 정의한다.

동영상 및 오디오embed

video, audio, source, track 요소는 HTML5에 새롭게 도입된 요소로, 동영상 및 오디오 컨텐츠를 삽입하고자 할 때 사용한다.
동영상 및 오디오 컨텐츠의 경우 기존 HTML에서는 플러그인에 의존적이었지만, video요소와 audio요소의 등장으로 더 이상 플러그인에 의존하지 않고 동영상 및 오디오 컨텐츠를 제공할 수 있다.
플러그인에 의존적이지 않기 때문에 CSS를 활용하여 디자인을 적용할 수 있고, 자바스크립트로 동영상 및 오디오 등을 제어할 수 있다.

video, audio요소를 사용하여 동영상 및 오디오 컨텐츠를 삽입하는 경우, 웹 브라우저가 지원하는 동영상 및 오디오 형식이 통일되어 있지 않다.
아직까지는 문제가 해결되지 않은 상황이다. 현재 HTML5에서 영상 및 음성 코덱 그리고 해당 컨텐츠의 컨테이너 조합 형식으로 H, 264, AAC, MP4와 Theora, Vorbis, OGG 등을
후보로 생각하고 있지만, 아직까지 웹 브라우저 제조사 간 이견으로 규정되어 있지 않다.

video 요소나 audio 요소에는 다양한 속성을 지정할 수 있다.
  • autoplay - 자동 재생 기능을 담당한다.
  • poster - 동영상 컨텐츠가 재생되기 전에 미리 표시하고 싶은 이미지를 삽입하고자 할 때 사용한다.
  • controls - 동영상이나 오디오의 재생, 정지 등의 인터페이스를 지정할 수 있다.
  • loop - 재생이 끝난 후 다시 재생할 것인지를 결정하는 논리 속성이다.
  • scr - 소스파일
  • preload - 웹 문서 로드 시 재생할 미디어를 미리 로드
  • mediagroup - 여러 개의 멀티미디어 컨텐츠 연결
  • muted - 음소거
  • crossorigin-CORS(Cross-Origin Resource Sharing) - 자격증명
  • width - 가로 크기
  • height - 세로크기


    • 비트맵 이미지canvas

    canvas 요소는 HTML5에 새롭게 도입된 요소로, 웹 페이지에 동적으로 이미지를 그릴 수 있는 기능이다.
    HTML5에서의 canvas 요소 지원은 독립적인 플러그인(플래시나 실버라이트 등) 없이도 애니메이션 등의 이미지 표현이 마크업에서 가능해졌다는 뜻이다.
    canvas 요소는 object 요소와 마찬가지로 시작 태그와 종료 태그 사이에 폴백(fallback) 컨텐츠를 지정하여 접근성을 보장하도록 하고 있다.
    canvas 요소에 동적으로 그림을 그리기 위해서는 자바스크립트를 사용해야 한다.

    이미지 맵map, area

    map 요소는 이미지 맵을 정의할 때 사용할 수 있다.
    map 요소 안에는 area 요소로 링크 영역 및 링크 정보 등을 지정한다.
    map 요소의 경우 name 속성은 필수 속성이므로 생략하면 안 된다.
    area 요소의 경우 img 요소와 마찬가지로 alt 속성에 대체 텍스트를 제공할 수 있다.
